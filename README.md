# 📱 Бэкенд мессенджера Elis Messenger

> 👋 **Привет! Это мой личный пет-проект (Pet Project).**  
> Я создала его самостоятельно исключительно в целях обучения, тренировки и прокачки своих хард-скиллов в разработке и автоматизации тестирования (QA Automation). На этом проекте я практикуюсь связывать бэкенд на Java с мобильной вёрсткой, настраивать базы данных и покрывать всё взрослыми, сложными автотестами. Проект постоянно дорабатывается и развивается! 🚀

Современный и высокопроизводительный REST API бэкенд для кастомного мобильного мессенджера. Построен на базе **Spring Boot 4**, **WebSockets (STOMP)** и **PostgreSQL**. Проект полностью защищен автоматическими тестами с использованием изолированной тестовой базы данных в оперативной памяти.

## 🚀 Архитектура и возможности API
* **Управление пользователями:** Эндпоинты для регистрации и входа (с автоматической очисткой пробелов и приведением к нижнему регистру), выгрузка списка контактов, а также ручка безопасного удаления профиля.
* **Чат в реальном времени:** Живой двухсторонний обмен сообщениями по протоколу STOMP через веб-сокеты.
* **Стандартизация REST:** Все ответы HTTP-контроллеров переведены на профессиональные рельсы `ResponseEntity` с четкими статус-кодами (`200 OK`, `201 Created`, `404 Not Found`).

## 🛠 Стек технологий
* **Ядро:** Java 21, Spring Boot 4.1.0, Spring Data JPA
* **Базы данных:** PostgreSQL (Рабочая БД), H2 Database (Изолированная тестовая БД в RAM)
* **Документация API:** Springdoc OpenAPI / Swagger UI (доступна при запуске по адресу `http://localhost:8080/swagger-ui/index.html`)
* **Утилиты:** Lombok

## 🧪 Автоматизация тестирования и контроль качества
В проекте настроен жесткий контроль качества кода — достигнуто **88% общего покрытия строк (Line Coverage)**, проанализированного через плагин **JaCoCo**. Покрытие ключевой бизнес-логики в контроллерах составляет **100%**.

* **Юнит-тестирование (Mockito):** Полная изоляция логики `UserController` и `MessageController`. Тяжелые слои репозиториев заменены «картонными муляжами» (моками), что гарантирует выполнение тестов за сотые доли секунды без реальных запросов в базу данных.
* **Интеграционное тестирование (H2 и MockMvc):** Проверка полной сборки приложения и его веб-контекста. Тесты симулируют реальный HTTP-трафик с помощью «виртуального Постмана» (`MockMvc`), при этом физически записывая тестовые данные в чистую временную схему базы данных H2 в оперативной памяти компьютера.

### Как запустить тесты и отчет JaCoCo локально:
Чтобы прогнать весь тестовый пакет и сгенерировать интерактивный отчет по покрытию, выполните в терминале команду:
```bash
./mvnw clean test
```
После успешного завершения сборки (`BUILD SUCCESS`) отчет в формате HTML можно открыть в браузере по пути: `target/site/jacoco/index.html`.
